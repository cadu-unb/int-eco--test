# ECO0019 — Introdução à Economia

Disciplina: **Economia**

## Arquivos JSON

- `json/ECO0019_P1.json` — Prova 1 (Capítulos 1-2)
- `json/ECO0019_P2.json` — Prova 2 (Capítulos 3-5)

Cada arquivo segue estrutura `meta` + `sections` + `questions`.

## Estrutura esperada

```json
{
  "meta": {
    "code": "ECO0019",
    "title": "Introdução à Economia - Prova 1",
    "subject": "Economia",
    "group": "Prova 1",
    "description": "Questões de revisão..."
  },
  "sections": [
    {
      "id": "cap-01",
      "title": "Dez princípios de economia",
      "questions": [...]
    }
  ]
}
```

## Tipos de questão usados

Todas as 4 categorias são suportadas:

1. **single_correct** — Uma única resposta certa
2. **true_false** — Verdadeiro ou falso
3. **multiple_correct** — Múltiplas respostas corretas
4. **single_incorrect** — Identificar alternativa incorreta

Veja `docs/prompt/` para templates de criação.

## Regras de correção

Todas as respostas são **arrays**, mesmo com uma alternativa:

```js
question.answer = ["a"]                    // single_correct
question.answer = ["true"]                 // true_false
question.answer = ["a", "c", "d"]         // multiple_correct
question.answer = ["c"]                    // single_incorrect
```

**Comparação normalizada:**
- Converter `["A"]` → `["a"]` (minúscula)
- Ordenar `["c", "a"]` → `["a", "c"]`
- Comparar arrays elemento a elemento

Resposta vazia (`[]`) é sempre **incorreta**.

## IDs globais

Formato:
```
ECO0019_P1.json::cap-01::q001
ECO0019_P1.json::cap-02::q008
ECO0019_P2.json::cap-03::q001
```

Usado para:
- Armazenar respostas do usuário
- Persistir favoritos em localStorage
- Navegar entre questões

## Exemplo: Questão com imagem

`command_image`, `justification_image` e `alternatives[].image` são **opcionais**. Caminho relativo à raiz: `img/ECO0019/arquivo.ext`. Aceitam **string** (uma imagem) ou **array** (várias imagens).

### Imagem única

```json
{
  "id": "q010",
  "type": "single_correct",
  "command": "Observe o gráfico de oferta e demanda abaixo. Qual ponto representa o equilíbrio de mercado?",
  "command_image": "img/ECO0019/oferta-demanda.png",
  "alternatives": [
    { "id": "a", "text": "Ponto A — interseção das curvas." },
    { "id": "b", "text": "Ponto B — máximo da curva de oferta." },
    { "id": "c", "text": "Ponto C — mínimo da curva de demanda." },
    { "id": "d", "text": "Ponto D — origem do eixo de preços." }
  ],
  "answer": ["a"],
  "justification": "O equilíbrio ocorre no ponto em que quantidade ofertada iguala quantidade demandada — a interseção das duas curvas.",
  "justification_image": "img/ECO0019/oferta-demanda-gabarito.png"
}
```

### Múltiplas imagens (array)

Use array quando o enunciado ou a justificativa precisar de mais de uma figura (ex: situação antes e depois, dois gráficos para comparar).

```json
{
  "id": "q012",
  "type": "single_correct",
  "command": "Compare os dois diagramas de fluxo circular abaixo e indique qual representa uma economia aberta.",
  "command_image": [
    "img/ECO0019/fluxo-fechado.png",
    "img/ECO0019/fluxo-aberto.png"
  ],
  "alternatives": [
    { "id": "a", "text": "O diagrama A — sem setor externo." },
    { "id": "b", "text": "O diagrama B — com exportações e importações." },
    { "id": "c", "text": "Ambos representam economia aberta." },
    { "id": "d", "text": "Nenhum dos dois representa economia aberta." }
  ],
  "answer": ["b"],
  "justification": "Economia aberta inclui transações com o exterior. O diagrama B mostra o setor externo (exportações e importações) ausente no diagrama A.",
  "justification_image": [
    "img/ECO0019/fluxo-aberto-destaque.png",
    "img/ECO0019/fluxo-aberto-legenda.pdf"
  ]
}
```

Array em `alternatives[].image` funciona da mesma forma:

```json
{ "id": "a", "text": "Curva com intercepto positivo.", "image": ["img/ECO0019/curva-a1.png", "img/ECO0019/curva-a2.png"] }
```

## Exemplo: Questão com tabela markdown

Tabelas no formato markdown são renderizadas automaticamente em `command`, `justification` e `alternatives[].text`. Separe a tabela do texto com uma linha em branco.

```json
{
  "id": "q011",
  "type": "single_correct",
  "command": "Com base na tabela de elasticidade-preço abaixo, qual bem tem demanda mais inelástica?\n\n| Bem | Elasticidade |\n| --- | ------------ |\n| Sal | 0,1 |\n| Carro | 1,5 |\n| Viagem aérea | 2,4 |",
  "alternatives": [
    { "id": "a", "text": "Sal" },
    { "id": "b", "text": "Carro" },
    { "id": "c", "text": "Viagem aérea" },
    { "id": "d", "text": "Todos igualmente inelásticos" }
  ],
  "answer": ["a"],
  "justification": "Elasticidade-preço próxima de zero indica demanda inelástica — a quantidade demandada varia pouco com o preço. Sal (0,1) é muito menos responsivo a preço que carro (1,5) ou viagem aérea (2,4)."
}
```

## Manutenção

### Ao adicionar questões

Checklist:

- [ ] ID único dentro da seção (ex: `q001`, `q002`, não repetir)
- [ ] `type` válido (`single_correct`, `true_false`, `multiple_correct`, `single_incorrect`)
- [ ] `command` claro e formal
- [ ] Cada alternativa tem `id` único (a, b, c, d, e...)
- [ ] `answer` é **sempre array** (ex: `["a"]`)
- [ ] Cada ID em `answer` existe em `alternatives`
- [ ] `justification` explica por que resposta está correta
- [ ] Se usar imagem: arquivo existe em `img/ECO0019/` e caminho está correto
- [ ] Se usar tabela markdown: bloco separado por linha em branco dupla (`\n\n`)
- [ ] Arquivo é JSON válido (sem trailing commas, aspas corretas)
- [ ] Arquivo adicionado a `json/index.json` (se novo arquivo)

### Validar JSON

```bash
# Usando jq (se instalado)
jq empty json/ECO0019_P1.json

# Ou use jsonlint.com no navegador
```

### Revisar distratores

Boas práticas para alternativas incorretas:

- **Erros conceituais comuns:** Não óbvios
- **Relacionadas ao tema:** Plausíveis
- **Tecnicamente precisas:** Exceto pela falha
- **Não absurdas:** Enganar quem estudou pouco, não quem não estudou

Exemplo ruim: "Economia é cor azul" ← óbvio demais

Exemplo bom: "Custo de oportunidade é o preço cobrado por um bem" ← parece correto mas ignora alternativas
